# **Data Analysis Training – Daily Diary (Day 9)**

**Date:** ___________

Today was the ninth day of my Data Analysis training. The session focused on **Object-Oriented Programming (OOP)** in Python and provided a brief overview of all its major concepts. I learned how OOP helps in developing organized, reusable, and maintainable programs by representing real-world entities as objects.

The session began with an introduction to **Classes and Objects**. I revised that a class is a blueprint used to create objects, while an object is an instance of a class containing attributes (data) and methods (functions). I practiced creating classes and objects and accessing their members.

Next, I reviewed the **`__init__` Method (Constructor)**. I learned that this special method is automatically executed whenever an object is created and is used to initialize object attributes with the required values.

The trainer then explained the four main principles of **Object-Oriented Programming**:

* **Encapsulation:** I learned how data and methods are combined into a single unit (class) and how access to data can be controlled to improve security and maintainability.
* **Abstraction:** I understood that abstraction hides unnecessary implementation details and exposes only the essential features of an object, making programs simpler to use.
* **Inheritance:** I revised how a child class can inherit the properties and methods of a parent class. The trainer briefly discussed different types of inheritance, including single, multilevel, multiple, hierarchical, and hybrid inheritance.
* **Polymorphism:** I learned that polymorphism allows the same method or function name to behave differently depending on the object or context. This increases flexibility and code reusability.

The session also covered the concepts of **Method Overriding** and **Method Overloading**. I learned that method overriding enables a child class to provide its own implementation of a method defined in the parent class, while method overloading in Python is generally achieved using default arguments or variable-length arguments since Python does not support traditional method overloading.

During the practical session, I implemented simple Python programs demonstrating classes, objects, constructors, inheritance, encapsulation, and polymorphism. These exercises helped me understand how OOP concepts work together to build well-structured applications.

Overall, today's session provided a comprehensive overview of Object-Oriented Programming in Python. I gained a better understanding of the core OOP principles and their practical applications. These concepts will help me write modular, reusable, and scalable programs, which are essential for software development and data analysis projects.
